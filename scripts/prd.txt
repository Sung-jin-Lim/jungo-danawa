# Jungo-Danawa: Product Requirements Document

## 1. Introduction

Jungo-Danawa is a web platform that provides an integrated solution for comparing prices of products across multiple online marketplaces in Korea. The platform focuses on helping users find the best deals by scraping data from second-hand marketplaces (Danggeun, Bunjang, Junggonara) and comparing them with retail prices from e-commerce platforms (Coupang).

### 1.1 Purpose
To create a unified search and comparison platform that helps users make informed purchasing decisions by transparently showing price differences between new and second-hand items.

### 1.2 Target Audience
- Budget-conscious consumers looking for the best deals
- Tech enthusiasts seeking price comparisons across platforms
- Regular shoppers who want to compare new vs. used product prices

## 2. Product Overview

### 2.1 Product Vision
Jungo-Danawa aims to be the go-to platform for price comparisons across Korean e-commerce and second-hand marketplaces, offering transparent pricing data and AI-powered product analysis.

### 2.2 Core Value Proposition
- Unified search across multiple platforms
- Real-time price comparisons between new and used products
- AI-powered analysis of tech product specifications and value
- Deal rating system for identifying the best offers

## 3. Key Features

### 3.1 Multi-Platform Search System
- Integrated search across Danggeun, Bunjang, Junggonara, and Coupang
- Standardized results display with source indication
- Filtering options by platform, price range, and keywords
- Sorting options by price (ascending/descending)

### 3.2 Price Comparison Engine
- Direct comparison between retail and second-hand prices
- Market price analysis showing percentage differences
- Visual indicators for good deals (below market value)
- Historical price tracking (future feature)

### 3.3 Product Detail Pages
- Comprehensive product information
- Multiple product images
- Market price analysis and comparison
- Similar product recommendations
- Original listing access via direct links

### 3.4 Tech Product Comparison Tool
- AI-powered analysis of tech product specifications
- Side-by-side comparison of multiple products
- Feature and value assessment
- Recommendation engine based on user preferences

### 3.5 Deal Rating System
- Algorithm-based rating of deals considering condition, price, and market value
- Visual indicators for exceptional offers
- Explanation of rating factors

## 4. Technical Requirements

### 4.1 Scraping System
- Real-time scraping from multiple platforms
- Handling of anti-scraping measures
- Proxy support for IP rotation
- Automated error recovery and retries
- Error reporting and monitoring

### 4.2 Data Processing
- Standardization of data from different sources
- Price parsing and normalization
- Image URL extraction and processing
- Product categorization
- Duplication detection and handling

### 4.3 User Interface
- Responsive design for all devices
- Fast loading search results
- Intuitive filtering and sorting
- Clean product detail display
- Easy comparison view for multiple products

### 4.4 Backend System
- RESTful API architecture
- MongoDB database for product storage
- Caching system for frequent searches
- Rate limiting and request throttling
- Authentication system (for future features)

### 4.5 AI Integration
- Product comparison analysis
- Deal value assessment
- Natural language search enhancement
- Recommendation engine

## 5. Tech Stack

### 5.1 Frontend
- React.js with Vite for fast development
- Material-UI for component library
- React Router for navigation
- State management with React hooks
- Axios for API requests

### 5.2 Backend
- Node.js with Express
- MongoDB for database
- Puppeteer for browser automation
- Cheerio for HTML parsing
- Caching system for optimization

### 5.3 Infrastructure
- Docker containerization
- CI/CD pipeline for automated deployment
- Monitoring and logging systems
- Proxy management for scraping

## 6. User Flows

### 6.1 Search Flow
1. User enters search query on homepage
2. System performs searches across all platforms
3. Results are displayed with filtering options
4. User can sort and filter results
5. User selects a product to view details

### 6.2 Comparison Flow
1. User selects multiple products for comparison
2. System generates side-by-side comparison
3. AI analysis provides insights on relative value
4. User can navigate to individual product pages

### 6.3 Product Detail Flow
1. User views comprehensive product information
2. Market analysis shows price comparison
3. Similar products are displayed
4. User can add to comparison or visit original listing
